# Prompt — Correção visual do sistema Vidalys Pay

Quero que você faça uma correção visual completa do sistema **Vidalys Pay**, um gerador de links de pagamento da marca Vidalys.

Use como referência os arquivos deste pacote:
- `01_identidade_visual_vidalys_pay.png`
- `02_icones_favicon_exportacao_vidalys_pay.png`
- `03_design_system_vidalys_pay.png`
- `04_guia_implementacao_handoff_vidalys_pay.png`

## Objetivo
Ajustar o sistema para que ele fique visualmente consistente, moderno, limpo, mobile-first, com aparência premium de fintech/SaaS, alinhado à identidade da Vidalys e focado em clareza, confiança, agilidade e usabilidade.

## Direção visual da marca
A marca Vidalys Pay deve transmitir:
- tecnologia
- confiança
- segurança
- fluidez
- rapidez
- controle
- crescimento

## Identidade visual obrigatória
Aplique corretamente:
- logotipo principal "Vidalys Pay"
- ícone reduzido da marca
- favicon e ícones de app
- paleta principal em azul profundo, azul elétrico, ciano e verde de apoio
- tipografia principal com visual moderno e limpo
- fundo claro com uso estratégico de azul escuro em elementos de destaque

## Ajustes visuais esperados
Revise e corrija:
1. cabeçalho da aplicação
2. logo no topo
3. sidebar/menu lateral
4. bottom navigation mobile
5. botões primários, secundários e destrutivos
6. estados de hover, foco, ativo e desabilitado
7. campos de formulário
8. cartões de resumo
9. badges de status de pagamento
10. toasts de notificação
11. estados vazios
12. loading states
13. histórico de links
14. telas mobile
15. espaçamentos
16. hierarquia tipográfica
17. consistência de bordas, sombras e ícones

## Regras de UI
- manter visual limpo e profissional
- usar bordas suaves
- usar sombras discretas
- evitar excesso de cores fortes ao mesmo tempo
- manter alto contraste e boa legibilidade
- priorizar experiência mobile sem perder boa visualização em desktop
- manter grid consistente
- reduzir ruído visual
- usar ícones simples e coerentes
- reforçar sensação de produto real e pronto para produção

## Componentes obrigatórios
Garanta que a interface final tenha:
- app bar com logo Vidalys Pay
- menu lateral ou navegação clara
- formulário para criar link de pagamento
- histórico de links
- perfil do usuário/vendedor
- cards de resumo
- badges de status: Criado, Aguardando, Pago, Falhou, Cancelado, Expirado, Reembolsado
- toasts de feedback
- empty state
- loading state
- sucesso de criação de link
- ação de reenviar via WhatsApp

## Cores sugeridas
Use como base:
- Azul profundo: #0A1B3D
- Azul elétrico: #2563FF
- Ciano: #00D4FF
- Verde de apoio: #00C896
- Fundo claro: #F2F5FA
- Cinza de apoio: #8792A6

## Entrega esperada
Quero que você:
1. analise visualmente a interface atual
2. liste os problemas visuais encontrados
3. proponha melhorias objetivas
4. reestruture a interface com base na identidade Vidalys Pay
5. descreva a nova organização visual por tela e por componente
6. se estiver gerando código, entregue HTML/CSS/Tailwind ou componentes reutilizáveis organizados
7. mantenha nomes de componentes claros e prontos para uso

## Tom do produto
O sistema deve parecer:
- confiável
- premium
- simples
- veloz
- moderno
- elegante sem frescura

Faça uma correção visual de ponta a ponta. Nada de deixar o sistema com cara genérica. Quero aparência forte de produto Vidalys Pay.
